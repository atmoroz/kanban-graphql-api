# Epic 13: Release & Deploy

**Мета:** Контроль виконання всіх епіків, створення README для проєкту та деплой API на сервер. Фінальний крок перед релізом.

---

## У scope

**Фічі:** перевірка готовності по епіках, документація в корені репозиторію, seed-скрипт для БД, викладка на хостинг.

**Use-cases:**

| ID | Дія |
|----|-----|
| UC-R1 | **Контроль виконання епіків** — пройтися по списку епіків (01–12) і переконатися, що критерії прийняття виконані або зафіксовані виключення/борг. За потреби оновити чек-листи та BACKLOG. |
| UC-R2 | **README** — створити або оновити файл `README.md` в корені проєкту: опис проєкту, як запустити локально (залежності, `.env`, команди), як запустити тести, посилання на GraphQL endpoint та за потреби на документацію (епіки, FEATURES). |
| UC-R3 | **Seed-скрипт** — скрипт, що заповнює БД початковими/демо-даними (користувачі, дошки, колонки, задачі, мітки, статуси, учасники). Запуск через `prisma db seed` або `npm run seed`; ідемпотентність або явна очистка перед заповненням (на вибір). |
| UC-R4 | **Деплой на сервер** — розгорнути застосунок на обраній платформі (VPS, Railway, Render, Fly.io, Vercel serverless тощо): збірка, змінні оточення, запуск процесу, доступ до API по URL. |

**Реалізація (high-level):**

- **Контроль епіків:** чеклист по кожному епіку (посилання на EPIC-01 … EPIC-12), позначки [ + ] по виконаних критеріях; коротка зводка в BACKLOG або в цьому епіку.
- **README:** розділи: назва та призначення проєкту; вимоги (Node, БД за наявності); встановлення (`npm install`); конфіг (`.env.example` або перелік змінних); скрипти (`npm run dev`, `npm test`, за наявності `prisma generate`/migrate, `prisma db seed`); як відкрити GraphQL (URL); посилання на `docs/` (епіки, уроки).
- **Seed-скрипт:** у `prisma/seed.ts` (або `seed.js`) — створення мінімального набору сутностей через Prisma Client (User, Board, Column, Task, Label, Status, BoardMember тощо); у `package.json` — `"prisma": { "seed": "..." }` та за потреби `"seed": "prisma db seed"`; у README — як і коли запускати seed (після першої міграції, для dev/staging).
- **Деплой:** обраний хостинг; налаштування env (DATABASE_URL, JWT_SECRET, EMAIL_PROVIDER тощо); команда старту (наприклад `node dist/index.js` після збірки або `ts-node`/`tsx` у dev); перевірка здоров'я (`/graphql` або health endpoint).

## Критерії прийняття

- [x] Проведено контроль виконання епіків 01–12: критерії позначені або задокументовані виключення/борг.
