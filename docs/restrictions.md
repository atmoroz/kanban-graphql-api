# Обмеження (Limits)

**Задача:** ввести ліміти на кількість сутностей у системі та повертати помилку при їх перевищенні.

---

## Мета

Обмежити зростання даних і навантаження: один користувач не може створити необмежену кількість дошок, а на одній дошці — колонок, задач і міток. При спробі перевищити ліміт API має повертати зрозумілу помилку (наприклад, з кодом `LIMIT_EXCEEDED` або `VALIDATION_FAILED`).

---

## Ліміти

| Константа | Значення | Застосування |
|-----------|----------|--------------|
| `MAX_BOARDS_PER_USER` | 5 | Кількість дошок, власником або учасником яких може бути один користувач |
| `MAX_COLUMNS_PER_BOARD` | 20 | Кількість колонок на одній дошці |
| `MAX_TASKS_PER_BOARD` | 200 | Кількість задач на одній дошці (сумарно по всіх колонках) |
| `MAX_LABELS_PER_BOARD` | 50 | Кількість міток на одній дошці |

Перевірка виконується **перед** створенням нової сутності (при виклику `createBoard`, `createColumn`, `createTask`, `createLabel` відповідно). Учасники дошки рахуються за членством (BoardMember), а не тільки за власником.

---

## Де перевіряти

- **Дошки:** при `createBoard` — порахувати дошки, де поточний користувач є власником або учасником (через BoardMember). Якщо вже >= 5 — помилка.
- **Колонки:** при `createColumn` — порахувати колонки по `boardId`. Якщо вже >= 20 — помилка.
- **Задачі:** при `createTask` — порахувати всі задачі по дошці (через column.boardId). Якщо вже >= 200 — помилка.
- **Мітки:** при `createLabel` — порахувати мітки по `boardId`. Якщо вже >= 50 — помилка.

---

## Формат помилки

При перевищенні ліміту повертати GraphQL-помилку з:

- **extensions.code:** наприклад `LIMIT_EXCEEDED` (або використовувати існуючий `VALIDATION_FAILED`, якщо в проєкті не вводять окремий код).
- **message:** human-readable message, e.g. «Limit reached: maximum 5 boards per user» (and similar for columns, tasks, labels).

Узгодити з форматом помилок з [EPIC-01](../epics/EPIC-01-foundation.md) (якщо є єдиний довідник кодів).

---

## Критерії прийняття

- [ ] Константи `MAX_BOARDS_PER_USER`, `MAX_COLUMNS_PER_BOARD`, `MAX_TASKS_PER_BOARD`, `MAX_LABELS_PER_BOARD` задані в одному місці (наприклад, конфіг або модуль `lib/limits.ts`).
- [ ] При створенні дошки (`createBoard`) перевіряється ліміт дошок на користувача; при перевищенні повертається помилка з кодом і повідомленням.
- [ ] При створенні колонки (`createColumn`) перевіряється ліміт колонок на дошку; при перевищенні повертається помилка.
- [ ] При створенні задачі (`createTask`) перевіряється ліміт задач на дошку; при перевищенні повертається помилка.
- [ ] При створенні мітки (`createLabel`) перевіряється ліміт міток на дошку; при перевищенні повертається помилка.
- [ ] Повідомлення про помилку явно вказує, який ліміт перевищено і яке значення допустиме (наприклад, «Limit reached: maximum 20 columns per board»).
- [ ] Ліміти не застосовуються до операцій оновлення/видалення — тільки до створення нових сутностей.

---

## Додатково

- Ліміти можна винести в змінні оточення (наприклад, `MAX_BOARDS_PER_USER`), щоб змінювати їх без перезбірки; за замовчуванням — вказані вище значення.
- За потреби додати тести: успішне створення на межі ліміту та помилка при спробі створити понад ліміт.
