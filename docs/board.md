IMPORTANT TECHNICAL CONSTRAINTS

- Database: PostgreSQL (Neon)
- ORM: Prisma
- NO mocks, NO in-memory data
- Use Prisma Client only
- Use prisma.$transaction for the entire seed
- Task ↔ Label is many-to-many via TaskLabel table
- Do NOT use task.labelIds
- Status is derived from column.statusId
- All IDs must be generated via Prisma (uuid)
- Content language: Ukrainian


CREATION ORDER (STRICT)

1. Demo User (optional, can be system user)
2. Board (PUBLIC)
3. Statuses (4)
4. Columns (4, each linked to Status)
5. Labels (6)
6. Tasks (8)
7. TaskLabel relations


EXPECTED OUTPUT

- A Prisma seed script
- File: prisma/seed.ts
- Uses PrismaClient
- Wrapped in a single prisma.$transaction
- Can be executed via: npx prisma db seed

## Примітки для seed / моків

- **Board:** одна публічна дошка з фіксованим (або константним) `id`, щоб у seed/моках можна було одразу створювати колонки, статуси, метки та задачі по `boardId`.
- **Статуси:** 4 шт. з `boardId` дошки, поля `name`, `order`, `color` заповнені.
- **Колонки:** 4 шт., кожна з `statusId` відповідного статусу, `position` 0..3.
- **Метки:** 6 шт. з `boardId`, `name`, `color`.
- **Задачі:** 8 шт.; розподіл по колонках (наприклад 2 в «Готово», 3 в «В роботі», 1 «На перевірці», 2 в «Беклог»); у кожній задачі заповнити `title`, `description`, `priority`, `dueDate` (де потрібно), `statusId`, `labelIds` (1–3 метки на задачу), `position` у межах колонки.
- Мова контенту: **українська** (назви колонок, статусів, меток, задач та описи).

Цей документ використовується як специфікація для наповнення демо-дошки в seed-скрипті та/або в моках при старті застосунку.
